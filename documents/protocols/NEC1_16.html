IRP notes: protocol NEC1_16

I found the documentation about a NEC protocol not present in "Interpreting Decoded IR Signals (v2.45)".
Because my TV (HITACHI CL32WF740AN) and related remote control (HITACHI CLE941) uses this protocol, I built a new IRP from documentation, and I called it NEC1_16
  • it uses dittos, like NEC1.
  • it as 2 bytes of payload and 2 control bytes.
  • my remote command times are very near to 562 (documentation says 562.5) and not to 564.

NEC1_16

IRP notation: {38.0k,562}<1,-1|1,-3>(16,-8,D:8,~D:8,F:8,~F:8,1,^110m,(16,-4,1,^110m)*)

NEC1

from: Interpreting Decoded IR Signals:
IRP notation: {38.0k,564}<1,-1|1,-3>(16,-8,D:8,S:8,F:8,~F:8,1,^108m,(16,-4,1,^108m)*)

note:
My Protocol names have same rule as for PHP variables:<lowletter|Upperletter|underscore><lowletter|Upperletter|digit|underscore>*
